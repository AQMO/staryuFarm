# IDEA 启动指南

## 环境要求
- JDK 17+
- Maven 3.5+（3.5.2 已验证兼容）
- MySQL 8.0+
- Apache Tomcat 9（如使用外部 Tomcat 部署）

## 第一步：配置 Maven 镜像（解决插件下载失败）

**方法一：使用项目自带的 settings.xml（推荐）**

IDEA → File → Settings → Build Tools → Maven：
- 勾选 `User settings file`，指向项目中的 `.mvn/settings.xml`
- 或复制 `.mvn/settings.xml` 到你本机的 `~/.m2/` 目录下

**方法二：手动修改你本机的 settings.xml**

找到 `~/.m2/settings.xml`（没有就新建），添加阿里云镜像：

```xml
<settings>
  <mirrors>
    <mirror>
      <id>aliyun-public</id>
      <mirrorOf>central</mirrorOf>
      <name>Aliyun Public Maven Mirror</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>
```

## 第二步：初始化 MySQL 数据库

```bash
mysql -u root -p < init-db.sql
```

或手动执行：
```sql
CREATE DATABASE IF NOT EXISTS staryu_farm DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
-- 然后执行 init-db.sql 中的建表语句
```

## 第三步：IDEA 导入项目

1. File → Open → 选择项目根目录（pom.xml 所在目录）
2. 等待右下角 Maven 导入进度完成
3. 如果提示 "Unresolved plugin"，点击 Maven 面板的 🔄 刷新按钮

## 第四步：修改数据库连接（如需要）

编辑 `src/main/java/com/staryu/config/DataSourceConfig.java`：

默认连接（MYSQL_URL 环境变量未设置时）：
```java
// URL:  jdbc:mysql://localhost:3306/staryu_farm?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=UTF-8
// 用户: staryu
// 密码: staryu123
```

也可以通过环境变量配置：
```bash
# MySQL 格式
export MYSQL_URL=mysql://root:yourpassword@localhost:3306/staryu_farm

# PostgreSQL 格式（如使用 Supabase/云数据库）
export PGDATABASE_URL=postgresql://user:password@host:port/dbname
```

## 第五步：启动项目（两种方式）

### 方式一：外部 Tomcat 部署（推荐生产环境）

#### 5.1.1 下载安装 Tomcat 9

从 [Tomcat 9 官网](https://tomcat.apache.org/download-90.cgi) 下载，解压到本地目录。

#### 5.1.2 在 IDEA 中配置 Tomcat

1. Run → Edit Configurations → 点击 "+" → 选择 "Tomcat Server" → "Local"
2. 配置 Tomcat Home：选择你解压的 Tomcat 目录
3. 在 "Deployment" 标签页：
   - 点击 "+" → "Artifact" → 选择 `staryu-web:war exploded`
   - Application context 设为 `/`（根路径）
4. 在 "Server" 标签页：
   - HTTP port 设为 `5000`
   - JRE 选择 17+
5. 点击 OK 保存

#### 5.1.3 启动 Tomcat

点击 IDEA 右上角的绿色三角形 ▶ 按钮，启动 Tomcat。

控制台输出 `Server startup in xxx ms` 即启动完成。

### 方式二：嵌入式 Tomcat 启动（开发调试用）

找到 `src/main/java/com/staryu/config/EmbeddedMain.java`，右键 → Run EmbeddedMain.main()

控制台输出 `Server started at http://localhost:5000` 即启动完成。

**注意**：嵌入式方式需要先构建 WAR 文件：
```bash
mvn clean package -DskipTests
# 然后解压 WAR 到 /tmp 目录运行
mkdir -p /tmp/staryu-web && cd /tmp/staryu-web
jar -xf /path/to/staryu-web/target/staryu-web.war
java -cp 'WEB-INF/lib/*:WEB-INF/classes' -Dcatalina.base=/tmp/staryu-web -Dserver.port=5000 com.staryu.config.EmbeddedMain
```

## 部署到外部 Tomcat（生产环境）

### 手动部署

1. 构建 WAR 文件：
   ```bash
   mvn clean package -DskipTests
   ```
2. 将 `target/staryu-web.war` 复制到 Tomcat 的 `webapps/` 目录
3. 如果需要根路径访问，将 WAR 重命名为 `ROOT.war`：
   ```bash
   cp target/staryu-web.war $TOMCAT_HOME/webapps/ROOT.war
   ```
4. 启动 Tomcat：
   ```bash
   $TOMCAT_HOME/bin/startup.sh    # Linux/Mac
   $TOMCAT_HOME\bin\startup.bat   # Windows
   ```
5. 访问 http://localhost:8080/ (默认端口8080，可在 server.xml 中修改)

### 修改 Tomcat 端口为 5000

编辑 `$TOMCAT_HOME/conf/server.xml`，将 Connector port 改为 5000：
```xml
<Connector port="5000" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" />
```

### 配置数据库连接

在启动 Tomcat 前设置环境变量：
```bash
# MySQL
export MYSQL_URL=mysql://root:password@localhost:3306/staryu_farm

# 或 PostgreSQL
export PGDATABASE_URL=postgresql://user:password@host:port/dbname
```

或在 `$TOMCAT_HOME/bin/setenv.sh`（Linux）/ `setenv.bat`（Windows）中设置：
```bash
#!/bin/bash
export MYSQL_URL=mysql://root:password@localhost:3306/staryu_farm
```

## 数据库支持

项目支持 MySQL 和 PostgreSQL 两种数据库，通过环境变量自动检测：

| 环境变量 | 数据库 | Hibernate 方言 |
|----------|--------|---------------|
| MYSQL_URL | MySQL | MySQLDialect |
| PGDATABASE_URL | PostgreSQL | PostgreSQLDialect |
| 均未设置 | MySQL (本地默认) | MySQLDialect |

## 常见问题

### 1. Unresolved plugin: maven-shade-plugin
- 本项目已改为 WAR 打包，不再使用 shade 插件
- 确保 Maven 镜像已配置（见第一步）
- 点击 Maven 面板 🔄 刷新

### 2. java: 错误: 不支持发行版本 17
- IDEA → File → Project Structure → Project SDK 选择 17
- IDEA → Settings → Build → Compiler → Java Compiler → Target bytecode version 设为 17

### 3. 数据库连接失败
- 确认 MySQL/PostgreSQL 服务已启动
- 确认 staryu_farm 数据库已创建
- 确认用户名密码正确（环境变量或 DataSourceConfig.java）

### 4. Tomcat 部署后 404
- 确认 WAR 文件已正确放在 webapps/ 目录
- 检查 Tomcat 日志：$TOMCAT_HOME/logs/catalina.out
- 如果 WAR 文件名为 staryu-web.war，访问路径为 http://localhost:8080/staryu-web/
- 如需根路径访问，重命名为 ROOT.war

### 5. IDEA Tomcat 配置找不到 Artifact
- 先执行 Maven 构建一次：`mvn clean package -DskipTests`
- 然后 IDEA → Project Structure → Artifacts → 点击 "+" → Web Application: Exploded → From Modules
- 回到 Run Configuration → Deployment → 添加 Artifact

### 6. 中文乱码
- 确保 Tomcat 的 server.xml 中 Connector 配置了 URIEncoding="UTF-8"
- 项目已配置 CharacterEncodingFilter 强制 UTF-8
